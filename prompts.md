# Prompts usados para o desafio

1. Quais as diferenças principais entre a arquitetura de um projeto em Vue 3 e um projeto em Next.js? Compare pontos de estrutura de pastas, roteamento, data fetching e deploy.
2. Para que serve o slug em aplicações web e como implementá-lo em rotas dinâmicas?
3. Como funciona o roteamento no Next.js? Explique rotas de páginas, rotas dinâmicas, catch-all e rotas aninhadas.
4. Tenho um projeto pessoal em Vue que segue uma estrutura organizada; é possível manter uma estrutura parecida ao migrar para Next.js? Quais adaptações seriam necessárias?
5. Aqui está minha ideia e um rascunho de código do frontend: faça uma análise e aponte pontos de melhoria (estrutura, componentes, estilos, acessibilidade e performance).
6. Como implementar responsividade no projeto seguindo boas práticas do Next.js (breakpoints, layout fluido, componentes reutilizáveis e ajustes de tipografia/spacing)?
7. Compare minha arquitetura atual com este projeto de referência e sugira refatorações: estrutura de pastas, separação de responsabilidades, padrões de componentes e organização de estilos.
8. Recebi um erro de hidratação no Next.js; explique as causas comuns e como corrigir quando o HTML renderizado no servidor não bate com o do cliente (ex.: uso de window, Date.now/Math.random, dados dinâmicos, locale de data, HTML inválido).
9. Primeira experiência deployando na Vercel: preciso de um passo a passo e dicas para configurar build e variáveis de ambiente.
10. Durante o seed do banco (Supabase) a conexão falhou por TLS; como ajustar a DATABASE_URL e usar NODE_TLS_REJECT_UNAUTHORIZED=0 temporariamente para rodar o prisma seed?
11. Explica rápido o atributo d em <path> de um ícone SVG (comandos M, c, l, Z) e sugira opções mais legíveis para usar com Tailwind/Twind.
12. Me explica detalhadamente como deixar a barra de menu fixa (sticky) ao rolar a página no Next.js: CSS necessário (position: sticky/fixed, top), ajustes de layout, z-index, espaçamentos e comportamento em mobile.
